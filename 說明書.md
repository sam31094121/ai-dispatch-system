# 兆櫃智能派工系統 — 完整使用說明書

---

## 一、系統簡介

**兆櫃智能派工平台**是一套專為業務團隊設計的智能管理系統，核心功能如下：

- 每日業績資料匯入與解析（支援三大平台：奕心、民視、公司自營）
- 六層嚴格稽核引擎，自動偵測數據異常，確保資料可信度
- 依業績排名自動分組派單，分為四大戰力梯隊
- 整合 人工智慧 技術（Google Gemini），輔助決策與文字產出
- 一鍵產出日報表、公告、派單通知，直送 LINE 群組格式

## 五、稽核引擎說明（六層防線）

稽核系統採用嚴格的六層規則，任一層未通過即進入 **停止模式（禁止派單）**：

| 規則代號 | 名稱 | 說明 |
|----------|------|------|
| C1 | 天地盤一致性 | 各平台報表總業績 必須等於 個人業績加總，差額容忍為零 |
| C2 | 成交卻沒金額 | 有派單成交筆數，但總業績為 0，視為漏填 |
| C3 | 追單/續單卻沒金額 | 有追續成交或續單金額，但總業績為 0 |
| C4 | 位數不合理 | 單筆業績超過 200 萬，或續單金額大於總業績 |
| C5 | 名字衝突 | 輸入的姓名不在 21 位合法人員名單內 |
| C6 | 累積數字倒退 | 累積型數字（通數/業績）不可倒退或無故歸零 |

> **停止模式**：稽核失敗時，系統禁止輸出名次、派單及任何業績推算。須修正全部異常後重新提交。

## 六、排名派工 — 四大梯隊

排名計算完成後，系統依業績自動將業務人員分入四組：

| 分組代號 | 組別名稱 | 派單策略 |
|----------|----------|----------|
| **A1** | 突破之刃 | 高單主力，優先接單高客單價案件 |
| **A2** | 獵鷹部隊 | 續單收割，專注現有客戶深度耕耘 |
| **B** | 磐石陣線 | 量單主軸，維持穩定出單量 |
| **C** | 破風新銳 | 補位成長，培育中的潛力人員 |

業績排名整合**三大平台**資料計算：

- 奕心專案
- 民視專案
- 公司自營產品

## 七、核心引擎說明

| 引擎名稱 | 功能 |
|----------|------|
| 稽核引擎 | 執行 C1~C6 六層規則，出具稽核報告 |
| 廣播引擎 | 將派單結果轉換為標準廣播訊息格式 |
| 高價值引擎 | 篩選高客單價客戶，優先排入 A1 梯隊 |
| 趨勢引擎 | 分析歷史業績趨勢，提供排名參考依據 |
| 格式引擎 | 將資料轉換為符合 LINE 群組的訊息格式 |

## 九、注意事項

1. 請確認已安裝 **Node.js**（建議版本 18 以上）
2. 系統需連線至 **Google Gemini 人工智慧** 服務，請確認網路暢通
3. 資料庫使用本機 **SQLite**，資料存放於專案目錄內，請定期備份
4. 稽核未通過時，**絕對不可強行派單**，需先修正所有異常紅單
5. 人員名單共 **21 位**，新增人員請先至系統設定更新合法名單，否則資料會被 C5 稽核攔截
