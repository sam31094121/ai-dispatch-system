# 【每日業績輸入系統｜前端對接欄位命名稿＋接口欄位對照表 V1】

**系統版本**：前端對接欄位命名稿＋接口欄位對照表 V1  
**文件用途**：提供前端工程師、後端工程師、測試工程師共同使用。  

## 🎯 文件目標
一、統一前端畫面欄位命名  
二、統一前端表單資料結構  
三、統一接口請求欄位名稱  
四、統一接口回傳欄位名稱  
五、避免欄位名稱混亂、骨牌效應、前後端對不起來  

---

## 🛑 一、前端命名最高原則

1. **資料庫欄位**：維持後端 `snake_case`。
   - 例如：`report_date`、`platform_name`、`total_revenue_amount`
2. **前端程式欄位**：統一使用 `camelCase`。
   - 例如：`reportDate`、`platformName`、`totalRevenueAmount`
3. **畫面顯示名稱**：統一使用中文。
   - 例如：報表日期、平台、當月總業績（扣退貨）
4. **前端不得自行發明新欄位名稱。**
5. **前端不得自行縮寫核心欄位。**
6. **所有列舉值、下拉選單、狀態列都必須共用固定字典集中管理。**
7. **中介轉換**：前端送出資料時，若後端要求 `snake_case`，必須在 API 接口層/攔截器統一轉換，不可在畫面層亂寫。
